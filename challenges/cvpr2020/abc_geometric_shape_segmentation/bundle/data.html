Data

We supply 3 datasets and a starting kit (with evaluation and file loading python code)

  1. Training datasets: 294k point clouds with ground truth segmentations in 3 varying 3D resolutions: 0.02 mm/point (high resolution), 0.05 mm/point (medium), and 0.125 mm/point (low). The spatial size of each CAD model has been selected such that the 75% longest feature lines in that model are sampled with at least 8 points at the low 0.125 mm 3D resolution (and denser in other resolutions). Each point has a binary label associated to it, which is set to 1 for points lying within a pre-specified distance to the feature line, and zero otherwise. Specifically, this distance is set to be equal to the 3D resolution of the point cloud (i.e., for high-resolution instances, points located within 0.02 mm to features lines are marked as 1). We provide 98k annotated samples per each of the above 3D resolutions, packed in chunks of 16k samples each.
    There are two versions of the training dataset: A textfile version (1 point cloud per file) and a python/numpy version (16k point clouds per file).

  2. Validation datasets: 16k point clouds without ground truth segmentation per each 3D resolution. See Files for download.
  3. Testing datasets: 16k point clouds without ground truth segmentation per each 3D resolution. The evaluation on this dataset will result in the final score. See Files for download.
